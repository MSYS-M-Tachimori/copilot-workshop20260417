# Pomodoro Timer 実装機能一覧

このドキュメントは、ポモドーロタイマーアプリケーションで実装する必要のある機能を整理したものです。

## 1. MVPで必須の機能

### 1.1 タイマー基本操作
- 作業セッション（25分）を開始できる
- 一時停止できる
- 再開できる
- リセットできる
- 残り時間を `MM:SS` 形式で表示できる

### 1.2 セッションモード管理
- `work`（作業）と `break`（休憩）を切り替えられる
- 完了時に状態が `completed` になる
- 次セッション準備のため `idle` に戻せる
- 不正な状態遷移（例: `idle` から一時停止）を防ぐ

### 1.3 正確なカウントダウン
- `setInterval` の回数依存ではなく、終了予定時刻との差分で残り時間を算出する
- タブが非アクティブでも大きくズレない

### 1.4 進捗の可視化（UI）
- 円形プログレスなどで進行率を表示する
- 今日の完了回数を表示する
- 今日の集中時間（秒を時間/分に変換）を表示する

### 1.5 セッション完了記録 API
- 完了セッションをサーバーへ送信して記録できる
- 最低限の入力項目（種別、開始/終了時刻、継続秒数、完了フラグ）を受け取る
- バリデーションエラー時に適切なレスポンスを返す

### 1.6 今日の進捗取得 API
- 当日の `completed_sessions` を返せる
- 当日の `focused_seconds` を返せる
- 画面初期表示時に取得して反映できる

### 1.7 永続化（SQLite）
- セッション履歴を保存できる
- 日次集計を取得できる
- 最低限のセッションレコード項目を保持できる（id, session_type, started_at, ended_at, duration_seconds, completed, created_at）

### 1.8 画面構成と静的配信
- Flaskで画面表示と静的ファイル配信ができる
- モックに沿ったUI（タイマー表示、開始/リセット、進捗カード）を提供できる

### 1.9 エラーハンドリング
- API通信失敗時にUIへ通知できる
- 想定外エラー時のサーバー側ハンドリングができる
- クライアント側で最低限の状態復旧ができる

### 1.10 テスト（最低限）
- ドメイン: 状態遷移、完了判定、不正入力拒否
- アプリケーション: 完了記録と日次集計
- API: 正常系とバリデーション異常系

## 2. 拡張候補（MVP後）

### 2.1 設定変更
- 作業時間・休憩時間を変更できる
- 設定を永続化できる

### 2.2 通知
- セッション完了時の音通知
- ブラウザ通知（権限管理含む）

### 2.3 統計
- 日/週/月の集中時間表示
- 連続達成日数の可視化

### 2.4 ユーザー対応
- 将来の `user_id` 対応を見据えたデータ設計

## 3. 推奨実装順
1. Flaskで画面表示と静的ファイル配信を整備
2. UIモックに沿ったHTML/CSS実装
3. フロントの状態管理・タイマー計算を実装
4. セッション完了APIを実装
5. SQLite保存と日次集計APIを実装
6. テスト（domain/application/API）を追加
7. 通知や設定画面などの拡張を段階的に追加
